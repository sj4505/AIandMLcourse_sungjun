# 삽질 기록

프로젝트별로 겪은 문제와 해결 과정을 기록합니다.

---

## budget-app — Google OAuth + Supabase + Polar 결제 연동 (2026-04)

### 1. `module.exports.config` 덮어쓰기
**증상**: webhook bodyParser 설정이 무시됨  
**원인**: `module.exports.config = {...}` 후 `module.exports = handler`로 전체를 덮어써서 config 날아감  
**해결**: `handler.config = {...}` 를 함수에 직접 붙이고 `module.exports = handler`

---

### 2. Polar webhook secret prefix
**증상**: 서명 검증 실패  
**원인**: Polar는 `polar_whs_` prefix를 쓰는데 코드가 `whsec_`만 처리  
**해결**: 정규식 `/^(whsec_|polar_whs_)/` 로 수정

---

### 3. Vercel production branch 불일치 (핵심 삽질)
**증상**: 코드를 아무리 고쳐도 Vercel에 반영 안 됨. 디버그 코드 넣어도 옛날 응답만 나옴. 무조건 200 반환하는 코드도 401 반환  
**원인**: Vercel `ai_and_ml` 프로젝트가 `main` 브랜치를 바라보고 있었는데 계속 `portfolio` 브랜치에만 푸시함  
**발견**: `TEST_V5` 코드가 배포 확인됐음에도 401 → 새 코드가 절대 실행 안 됐다는 것 확인  
**해결**: Vercel API로 `portfolio` 브랜치 커밋을 production에 직접 배포 트리거  
**교훈**: 새 배포 후 코드 변화가 없으면 **branch 설정부터 확인**

---

### 4. Vercel 프로젝트 실수로 새로 생성
**증상**: `budget-app`이라는 환경변수 없는 프로젝트가 생겨 혼란  
**원인**: CLI 배포 테스트 중 의도치 않게 새 프로젝트 생성  
**해결**: Vercel API로 잘못 만들어진 프로젝트 삭제

---

### 5. HMAC 서명 직접 구현 vs 공식 SDK
**증상**: secret, body, timestamp 모두 정상인데 서명만 계속 불일치  
**원인**: 직접 구현한 HMAC 검증이 Polar 내부 구현과 미묘하게 다름  
**해결**: `@polar-sh/sdk` 의 `validateEvent()` 공식 SDK로 교체  
**교훈**: 암호화/서명 검증은 공식 SDK 있으면 직접 구현하지 말 것
